FALTANTES PARA WEB FUNCIONAL - EXITO AZUL
Fecha de actualizacion: 2026-04-02

OBJETIVO
Pasar de una web desplegada a un producto funcional y seguro para usuarios reales.

AVANCE ACTUAL (YA IMPLEMENTADO)
[OK] Persistencia real en Firestore para secciones e imagenes (por usuario)
[OK] Alta de seccion real en Firestore
[OK] Favorita on/off real en Firestore
[OK] Eliminacion real de imagen en Firestore + Storage
[OK] Subida real de imagen a Firebase Storage
[OK] Metadatos de imagen en Firestore (fileName, sectionId, isFavorite, storagePath, downloadUrl)
[OK] Naming automatico de imagenes: nombre_seccion_numero
[OK] Ruta "/" protegida por auth
[OK] Ruta compartida "/s/:token" publica

PENDIENTE PRIORIDAD 1 (BLOQUEANTE)
1) Registro real con Firebase Auth
- Conectar /register a createUserWithEmailAndPassword.
- Opcional recomendado: login email/password ademas de Google.

2) Vista compartida real (sin mock)
- Reemplazar SharedGalleryPage con datos reales por token.
- El token debe resolver una imagen o seccion puntual, no datos globales.

3) Links temporales en backend (obligatorio para seguridad real)
- createShareLink en Cloud Functions (token fuerte + expiracion + ownerId + targetType + targetId).
- Guardar links en Firestore (shareLinks).

PENDIENTE PRIORIDAD 2 (SEGURIDAD Y CONTROL)
4) Revocacion manual de links
- Function revokeShareLink.
- El usuario dueńo debe poder invalidar un link antes de su vencimiento.

5) Validacion de links en backend
- Token existente.
- No expirado.
- No revocado.
- Owner y recurso validos.

6) Reglas de seguridad finales
- Firestore rules por uid ownership.
- Storage rules por uid + validacion de tipo y tamano.
- Pruebas de acceso cruzado entre usuarios (debe fallar).

PENDIENTE PRIORIDAD 3 (ROBUSTEZ)
7) Validaciones funcionales
- Limite de tamano por imagen.
- Tipos MIME permitidos.
- Limite de volumen por usuario (control de costos).

8) Manejo de errores UX
- Mensajes especificos por caso (auth, network, permisos, cuota).
- Estados de carga y vacio en todas las vistas.

9) Auditoria y observabilidad
- Logs en Cloud Functions (creacion/revocacion/resolucion de links).
- Alertas basicas de fallos.

PENDIENTE PRIORIDAD 4 (GO-LIVE)
10) Entornos y despliegue
- Separar dev/staging/prod.
- Variables .env por ambiente.
- Revisar cache headers.

11) Dominio y legal
- Dominio custom (si aplica).
- Politica de privacidad final (texto legal real).
- Terminos y condiciones.

SIGUIENTE ORDEN RECOMENDADO

Sprint 3:
- Links temporales con backend + revocacion + vista compartida real por token
